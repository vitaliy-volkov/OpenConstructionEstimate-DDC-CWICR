# Региональные прайс-листы России (RU) — Pipeline

Цель: расширять RU-базу (`DDC CWICR`) по городам РФ на основе присылаемых прайсов, чтобы затем подключать эти города в ERP.

## 1) Что присылаешь ты

Поддерживаемые входные форматы:
- `.xlsx` (предпочтительно)
- `.csv`

Минимально нужные поля (любые названия колонок допустимы, я сопоставлю):
- `resource_code` **или** (`name` + `unit`)
- цена: `price_avg` или `price_median` (желательно также `price_min`, `price_max`)
- `currency` (для РФ обычно `RUB`)
- город (например, `Москва`, `Казань`, `Екатеринбург`)
- дата/период прайса

## 2) Нормализация и контроль качества

Перед добавлением в базу:
1. Нормализация единиц (`м2`, `м3`, `шт`, `т`, `кг` и т.д.).
2. Приведение цен к числам (без пробелов и валютных символов).
3. Дедупликация строк по ключу:
   - приоритет: `resource_code`
   - fallback: `name + unit`
4. Проверка аномалий:
   - нулевые/отрицательные цены
   - выбросы относительно RU_STPETERSBURG
   - пустые ключевые поля

## 3) Правило маппинга в DDC CWICR

Базовый справочник для слияния:
- `RU___DDC_CWICR/DDC_CWICR_RU_STPETERSBURG_Catalog.csv`

Ключи слияния:
1. `resource_code`
2. fallback: (`name`, `unit`)

Для каждой совпавшей позиции обновляются:
- `price_avg`
- `price_min`
- `price_max`
- `price_median`
- `currency`

## 4) Выходные артефакты по каждому городу

Именование города: `RU_<CITY>` в верхнем регистре, латиницей и `_`.
Пример: `RU_MOSCOW`, `RU_KAZAN`, `RU_EKATERINBURG`.

Минимальный результат:
- `RU___DDC_CWICR/DDC_CWICR_RU_<CITY>_Catalog.csv`

Плановый полный пакет (после подготовки полного набора):
- `RU___DDC_CWICR/RU_<CITY>_workitems_costs_resources_DDC_CWICR.parquet`
- `RU___DDC_CWICR/RU_<CITY>_workitems_costs_resources_DDC_CWICR_SIMPLE.xlsx`
- `RU___DDC_CWICR/RU_<CITY>_workitems_costs_resources_DDC_CWICR_FORMATTED.xlsx`
- `RU___DDC_CWICR/RU_<CITY>_workitems_costs_resources_EMBEDDINGS_3072_DDC_CWICR.snapshot`

## 5) Подключение в ERP

После подготовки города:
1. Добавляем `RU_<CITY>` как доступный регион в ERP.
2. Подключаем каталог/расценки к эндпоинтам импорта/поиска.
3. Обновляем векторный слой (Qdrant snapshot/коллекция).
4. Проверяем сценарии в UI:
   - импорт базы
   - выдача статуса vector
   - поиск расценок по региону

## 6) Политика публикации

- Для пользователей ERP загрузка региональных баз — **бесплатно**.
- Каждое обновление города фиксируется отдельным changelog-блоком (дата, источник, покрытие, качество).
